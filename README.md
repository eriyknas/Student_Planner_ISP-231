# Лабораторная работа №15-16. Navigation in Jetpack Compose
## Описание приложения
Приложение разработано для управления учебными дисциплинами. Оно позволяет студентам отслеживать список предметов, а также управлять профилем и настройками приложения.
## Список реализованных экранов
- Главный экран
- Экран деталей
- Экран профиля
- Экран настроек
- Экран расписания
## Используемые технологии
- Kotlin
- Jetpack Compose
- Navigation Compose
## Схема навигации
- Главный экран - `home`
- Экран деталей - `details`
- Экран профиля - `profile`
- Экран настроек - `settings`
- Экран расписания - `raspisanie`

## Контрольные вопросы
1. 
- navController — контроллер навигации, управляет переходами.
- rememberNavController() — создаёт NavController. remember сохраняет
   его при рекомпозиции.
2. Обязательные параметры всегда должны быть в маршруте, иначе навигация не сработает. Опциональные параметры могут отсутствовать и имеют значение по умолчанию.
3.  Sealed class гарантирует, что компилятор знает о всех возможных вариантах экранов.
4. 
- Back Stack — это стек (LIFO - Last In First Out) экранов, по которым переходил пользователь.
- popBackStack() — возврат назад (убирает текущий экран из стека).
5. startDestination определяет, какой экран будет показан первым при запуске приложения.
6. Приложение вызовет ошибку во время выполнения. Для обработки таких ситуаций можно использовать кастомный NavHost с проверками или ловить ошибки в NavController.
7. 
- launchSingleTop - предотвращает создание дублирующих экземпляров одного экрана в стеке. 
- Проблемы - без него многократный клик на "Профиль" добавит несколько экземпляров Profile в стек, что запутает пользователя.
